# JavaScript Best Practice Guide
---
### 1. Используйте `const` для объявления переменных, избегайте `var`.
Это гарантирует, что вы не сможете переопределять значения, т.к. это может привести к ошибкам и к усложнению понимания кода.

// плохо
``` js
var a = 1;
var b = 2;
```
// хорошо
``` js
const a = 1;
const b = 2;
```
---
### 2. Если вам необходимо переопределять значения, то используйте `let` вместо `var`.
Область видимости `let` — блок, у `var` — функция.

// плохо
``` js
var count = 1;
if (true) {
  count += 1;
}
```
// хорошо
``` js
let count = 1;
if (true) {
  count += 1;
}
```
---
### 3. В первую очередь группируйте `const`, а затем `let`.
Это полезно и удобно, когда в будущем вам понадобится создать переменную, зависимую от предыдущих.

// плохо
``` js
let i;
const items = getItems();
let dragonball;
const goSportsTeam = true;
let len;
```
// хорошо
``` js
const goSportsTeam = true;
const items = getItems();
let dragonball;
let i;
let length;
```
Такая запись намного упрощает восприятие и чтение кода.

---
### 4. Не создавайте цепочки присваивания переменных.
// плохо
``` js
(function example() {
  let a = b = c = 1;
}());
```
JavaScript интерпретирует это, как `let a = ( b = ( c = 1 ) );` - ключевое слово `let` применится только к переменной `a`, а переменные `b` и `c` станут глобальными.

// хорошо
``` js
(function example() {
  let a = 1;
  let b = a;
  let c = a;
}());
```
Это же правило действительно и для `const`.

---

### 5. Никогда не объявляйте числовые, строковые или логические объекты.

// плохо
``` js
var x = new String("John");
```
// хорошо
``` js
var y = "John";
```
`(x === y)`  // это ложь, т.к. `x` - объект, а  `y` - строка. 

---
### 6. Разделяйте операторы пробелами.
// плохо
``` js
const x=y+5;
```
// хорошо
``` js
const x = y + 5;
```
Такая запись намного упрощает восприятие/чтение кода.

---

### 7. Используйте === Сравнение.
// плохо
``` js
0 == "";        // true
1 == "1";       // true
1 == true;      // true
```
// хорошо
``` js
0 === "";       // false
1 === "1";      // false
1 === true;     // false
```
Т.к. `==` oператор сравнения всегда преобразует (в соответствующие типы) перед сравнением, а `===` оператор принудительно сравнивает значения и тип.

---
### 8. Используйте сокращения для булевских типов, а для строк и чисел применяйте явное сравнение.
// плохо
``` js
if (isValid === true) {
  // ...
}

if (name) {
  // ...
}

if (collection.length) {
  // ...
}
```
// хорошо
``` js
if (isValid) {
  // ...
}

if (name !== '') {
  // ...
}

if (collection.length > 0) {
  // ...
}
```
---

### 9. Остерегайтесь автоматического преобразования типов.
// пример
``` js
var x = "Hello";     // typeof x строка
x = 5;               // меняет typeof x на число
```
Т.к. числа могут быть случайно преобразованы в строки или NaN. JavaScript слабо типизирован и переменная может изменять свой тип данных. При выполнении математических операций JavaScript может преобразовывать числа в строки:
``` js
var x = 5 + 7;       // x.valueOf() : 12,  typeof x число
var x = 5 + "7";     // x.valueOf() : 57,  typeof x строка
var x = "5" + 7;     // x.valueOf() : 57,  typeof x строка
var x = 5 - 7;       // x.valueOf() : -2,  typeof x число
var x = 5 - "7";     // x.valueOf() : -2,  typeof x число
var x = "5" - 7;     // x.valueOf() : -2,  typeof x число
var x = 5 - "x";     // x.valueOf() : NaN, typeof x число 
```
---
### 10. Используйте понятные имена переменных и функций.
Код гораздо легче читать, когда при его написании используют понятные, описательные имена функций и переменных.

// плохо
``` js
function avg (a) {
  let s = a.reduce((x, y) => x + y);
  return s / a.lenght;
}
```

Его читабельность значительно улучшится, если использовать в нём понятные имена переменных и функций, отражающие их смысл.

// хорошо
``` js
function averageArray (array) {
  let sum = array.reduce((number, currentSum) => number + currentSum);
  return sum, array.length;
}
```


Не стремитесь к минимализму при написании текстов программ. Используйте полные имена переменных, которые тот, кто будет работать с вашим кодом в будущем, легко сможет понять.
